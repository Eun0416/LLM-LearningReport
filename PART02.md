# 용어 정리

### 프롬프트 엔지니어링 (Prompt Engineering)
- 생성형 AI와의 대화에서 원하는 결과를 얻기 위해 질문(프롬프트)을 설계하고 최적화하는 과정. 대화 질문 설계 또는 대화 질문 최적화라고도 함.

### 퓨샷 러닝 (Few-shot Learning)
- AI 모델이 제한된 수의 예시(few shots)만을 학습하여 새로운 개념이나 작업을 수행하는 학습 방식.

### 퓨샷 프롬프팅 (Few-shot Prompting)
- 퓨샷 러닝 능력을 활용하여 AI에게 원하는 결과에 대한 몇 가지 예시를 프롬프트에 포함시켜 답변의 정확도를 높이는 기술.

### 제로샷 프롬프팅 (Zero-shot Prompting)
- AI에게 아무런 예시 없이 원하는 결과를 직접 요청하는 프롬프팅 방식.

### 원샷 프롬프팅 (One-shot Prompting)
- AI에게 원하는 결과에 대한 단 하나의 예시를 프롬프트에 포함시켜 답변을 유도하는 방식.

### 투샷 프롬프팅 (Two-shot Prompting)
- AI에게 원하는 결과에 대한 두 개의 예시를 프롬프트에 포함시켜 답변을 유도하는 방식.

### 대화 질문 구성 요소
| 유형 (Type) | 설명 (Description) |
| - | - |
| 목적 (Objective) | 프롬프트 질문을 통해 얻고자 하는 최종 목표 또는 결과 |
| 맥락 (Context) | 질문과 관련된 배경 정보, 상황, 추가적인 세부 사항 |
| 역할 (Role, Persona) | AI 모델에게 부여하는 특정 역할 또는 관점 |
| 청중 (Audience) | 질문의 답변을 이해하고 활용할 대상 |
| 예시 (Examples) | AI에게 원하는 답변의 형식, 내용, 스타일을 명확히 보여주는 구체적인 사례 |
| 형식 (Format) | 답변 결과물의 구조 또는 형태 (ex: 표, 목록, 문단) |
| 키워드 (Keywords) | 답변에 반드시 포함되어야 하는 핵심 단어나 개념 |
| 제한 (Limitations) | 답변의 길이, 단어 수, 특정 내용 포함 여부 등에 대한 제약 조건 |
| 어조 (Tone) | 답변의 전반적인 분위기 또는 스타일 (ex: 친절한, 전문적인) |
| 부정 (Negation) | 답변에서 제외해야 하는 특정 내용이나 조건 |
